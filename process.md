# 📘 프로젝트 진행 흐름 (process.md)

> 에브리타임 강의평으로 **"수강신청 AI 비서"** 를 만든 과정을, 처음 보는 사람도 이해하도록 정리했습니다.

---

## 0. 한 줄 요약

우리 학교(가천대) 5과목의 **실제 강의평을 직접 수집**해서, **"이 교수 학점 잘 줘?"** 한 줄이면
AI가 **근거(출처)와 함께** 비교·추천해 주는 **채팅 웹앱**을 만들었습니다. 핵심 기술은 **RAG**입니다.

---

## 1. RAG가 뭔데? (3분 이해)

일반 챗봇(ChatGPT 등)은 우리 학교 강의평을 모릅니다. 모르면 그럴듯하게 **지어냅니다(환각)**.
RAG는 **"오픈북 시험"** 방식입니다.

```
질문: "모바일프로그래밍 학점 잘 주는 교수?"
   ├─ ① 검색(Retrieval): 강의평 DB에서 관련 리뷰를 찾아옴      ← 오픈북 펼치기
   └─ ② 생성(Generation): 찾아온 리뷰'만' 근거로 답을 만듦      ← 펼친 페이지 보고 답쓰기
```
> 모델 재학습이 아닙니다. 자료를 찾아 보여주고 답하게 하는 구조라, **데이터만 바꾸면 우리 학교용**이 됩니다.

## 2. 최종 시스템 한눈에

```
[수집]                    [정제/인덱싱]              [검색·생성]            [서비스]

에타 강의평  ──크롤러──▶ reviews_real.csv ──임베딩──▶ 벡터 저장소 ──RAG──▶ Streamlit
(가천대 5과목)  Playwright   (187건)         LSA/numpy      검색+필터+Claude   채팅 웹앱(공개 URL)
```

| 단계 | 파일 | 한 일 |
|------|------|-------|
| 수집 | `everytime_crawler.py` | Playwright로 5과목 강의평 187건 크롤링 |
| 정제 | (크롤러 내장) | 중복·외국인반/공학인증/영어강의 제외, 과목명 통일 |
| 인덱싱 | `embeddings.py`, `ingest.py` | 문장 → 의미 벡터 → 벡터 저장소(`store/`) |
| 검색 | `vectorstore.py`, `rag_vectordb.py` | 코사인 최근접 top-k + 교수/과목 필터 |
| 생성 | `rag_vectordb.py` | Claude API(없으면 오프라인 요약) |
| 서비스 | `app.py` | Streamlit 채팅 UI, 공개 URL 호스팅 |

## 3. 진행 흐름 (스토리)

### 1단계 — 목표 정의 (기획서 3.1~3.4)
| 번호 | 기능 | 쉽게 |
|------|------|------|
| 3.1 | 의미 검색 | 질문과 관련된 강의평 찾기 |
| 3.2 | 속성 필터 | "이 교수/과목만" 좁혀 보기 |
| 3.3 | 출처 표기 | `(가천대 모바일프로그래밍 26년 1학기 평점 5점 리뷰 참고)` 자동 |
| 3.4 | 환각 방지 | 자료 없으면 "데이터 없음" 거부 |

### 2단계 — 우리 학교 데이터 직접 수집
- 처음엔 공개 샘플 데이터로 뼈대를 만들었지만, **우리 학교 5과목 실제 데이터**로 가기로 결정.
- `everytime_crawler.py`(Playwright)로 로그인 → 과목 검색 → 강의평 탭에서 수집 → **187건**.
- 🔍 트러블슈팅:
  - 에타는 **Vue SPA** → 요소가 늦게 렌더 → "**기다렸다 찾기**" 로직 추가.
  - 강의평은 **개요가 아니라 `?tab=article` 탭**에 있음 → 탭 URL로 직접 이동.
  - 검색→클릭 방식이 멈춤 → **강의 URL 수집 후 직접 이동**으로 변경.
  - 외국인반/공학인증/영어강의 **제외**, 강의당 **최대 10건**, 과목명 **통일**.

### 3단계 — 임베딩 & 벡터 저장소
- **임베딩** = 문장을 "의미 좌표(숫자 벡터)"로 변환 → 글자가 안 겹쳐도 뜻으로 검색.
- 🔍 트러블슈팅: ChromaDB·onnx·torch가 **환경에서 네이티브 충돌(세그폴트)** →
  **순수 numpy 벡터 저장소**로 같은 기능을 직접 구현(의존성 0). 인터페이스가 같아 추후 교체 가능.

### 4단계 — RAG 서비스 (`rag_vectordb.py`)
```
질문 → ① 필터(교수/과목) → ② 의미검색 top-k → ③ 프롬프트(출처/환각방지) → ④ 생성
```
- ④ 생성: `ANTHROPIC_API_KEY` 있으면 **Claude**, 없으면 **규칙기반 오프라인 요약**.
- 출처에 **수강학기**까지 포함(예: `26년 1학기 수강자`).

### 5단계 — 채팅 웹앱 & 배포 (`app.py`)
- **Streamlit 채팅 UI**: 과목 필터, 참고 리뷰 수, 출처 펼치기.
- **Streamlit Community Cloud**로 GitHub 연동 **공개 URL 호스팅**.
- API 키는 **Secrets**로 안전 관리(코드/깃 미저장). 키 없으면 오프라인으로 동작.

## 7. 용어 미니 사전
| 용어 | 쉽게 |
|------|------|
| RAG | 오픈북 시험형 AI — 찾아서(검색) 답하기(생성) |
| 임베딩 | 문장을 의미 좌표(숫자)로 변환 |
| 벡터DB | 그 좌표들을 저장·최근접 검색하는 창고 |
| LSA | TF-IDF를 의미 축으로 압축한 가벼운 임베딩 |
| 환각 | AI가 모르는 걸 지어내는 현상 |
| SPA | 자바스크립트로 화면을 그리는 웹앱(에타) |

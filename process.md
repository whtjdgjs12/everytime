# 📘 프로젝트 진행 흐름 (process.md)

> 에브리타임 강의평으로 **"수강신청 AI 비서"** 를 만든 과정을, 처음 보는 사람도 이해하도록 정리한 문서입니다.

---

## 0. 한 줄 요약

학생들이 강의평 수백 개를 일일이 안 읽어도, **"이 교수 학점 잘 줘?"** 한 줄만 물으면
AI가 **실제 강의평을 근거로** 비교·요약해 주는 서비스를 만들었습니다.
핵심 기술은 **RAG(검색 증강 생성)** 입니다.

---

## 1. 그래서 RAG가 뭔데? (3분 이해)

일반 챗봇(ChatGPT 등)은 우리 학교 강의평을 모릅니다. 모르면 **그럴듯하게 지어냅니다(환각).**

RAG는 이렇게 동작합니다 — **"오픈북 시험"** 이라고 생각하면 됩니다.

```
질문: "홍길동 교수 학점 잘 줘?"
   │
   ├─ ① 검색(Retrieval) : 강의평 DB에서 질문과 관련된 리뷰 몇 개를 찾아온다  ← 오픈북에서 페이지 펼치기
   │
   └─ ② 생성(Generation): 찾아온 리뷰'만' 근거로 답을 만든다               ← 펼친 페이지 보고 답쓰기
```

> **학습(training)이 아닙니다.** 모델을 새로 훈련하는 게 아니라, *있는 자료를 찾아서 보여주고 답하게* 하는 구조입니다.

## 3. 전체 진행 흐름 (스토리)

### 1단계 — 기획서 읽기
`project_proposal.md` 에 적힌 4가지 필수 기능을 목표로 잡았습니다.

| 번호 | 기능 | 쉽게 말하면 |
|------|------|------------|
| 3.1 | 의미 검색 | 질문과 관련된 리뷰 찾아오기 |
| 3.2 | 속성 필터 | "이 교수/이 과목만" 좁혀서 보기 |
| 3.3 | 출처 표기 | 답변 끝에 `(강대 OOO 평점 5점 리뷰 참고)` 자동으로 붙이기 |
| 3.4 | 환각 방지 | 자료 없으면 솔직하게 "데이터 없어요" 라고 거부 |

### 2단계 — 실제 데이터 정리 (`build_dataset.py`)
6개 학교 강의평 CSV(각 199건)를 하나로 합쳤습니다.

```
강대·건대·광운·숭실·한성·한신 (총 1,194건)
   │ 통합 + 정제(빈값/짧은 리뷰 제거) + 중복 제거
   ▼
reviews_real.csv  (995건)
```

> 🔍 **중간 발견 1**: `한성에타.csv` 가 `광운에타.csv` 와 **199건 전부 똑같았습니다**(크롤링 중복).
> 중복 제거가 알아서 걸러내서 → 최종 **995건**.

### 3단계 — 임베딩 & 벡터DB (`embeddings.py`, `vectorstore.py`, `ingest.py`)

**임베딩** = 문장을 "의미 좌표(숫자 벡터)"로 바꾸는 것.
비슷한 뜻의 문장은 가까운 좌표를 갖게 되어, **글자가 안 겹쳐도 의미로 검색**됩니다.

```
"학점 잘 줌"  →  [0.21, -0.08, 0.55, ...]
"성적 후하게 줌" →  [0.19, -0.05, 0.58, ...]   ← 좌표가 가까움 = 비슷한 뜻
```

이 좌표들을 모아두고 빠르게 "가까운 것 찾기"를 하는 창고가 **벡터DB** 입니다.

> 🔍 **중간 발견 2 (트러블슈팅)**: 원래 유명한 벡터DB인 **ChromaDB** 를 쓰려 했지만,
> 이 PC에서 ChromaDB·onnxruntime·PyTorch 같은 **네이티브 라이브러리가 전부 충돌(세그폴트/DLL 오류)** 났습니다.
> → 그래서 **똑같은 기능을 순수 Python(numpy)으로 직접 구현**했습니다.
> (임베딩 저장 → 코사인 거리로 최근접 검색 → 메타데이터 필터 → 파일로 영속화)
> 결과는 동일하고, 환경이 정상화되면 `vectorstore.py` 만 Chroma로 갈아끼우면 됩니다.

### 4단계 — RAG 서비스 (`rag_vectordb.py`)
질문이 들어오면 4단계를 거칩니다.

```
[질문] "컴퓨터프로그래밍 꿀강 추천"
   │
   ▼ ① 필터(3.2)   과목="컴퓨터프로그래밍" 인 리뷰로 후보 좁히기
   ▼ ② 검색(3.1)   질문을 임베딩 → 벡터DB에서 의미상 가까운 top-k
   ▼ ③ 조립(3.3/3.4) 찾은 리뷰를 근거로 묶고 "지어내지 말고 출처 표기해" 규칙 결합
   ▼ ④ 생성        Gemini 키 있으면 AI 문장 / 없으면 규칙기반 요약
   │
   ▼
[답변 + 출처]  예: "이영석 교수 평균 5.0/5 (강대 컴퓨터프로그래밍 평점 5점 리뷰 참고)"
```

### 5단계 — 강의계획서 PDF 보강 (`build_syllabi.py`)
강의평엔 없는 정보(평가 비중·교재·수업 목표)를 PDF 강의계획서에서 뽑아 보강했습니다.

```
temp_pdf/*.pdf  →  pdfminer로 한글 추출  →  중복 제거  →  450자씩 청킹
                →  syllabi.csv  →  리뷰와 '같은 벡터DB'에 함께 적재
```

이러면 한 질문이 **리뷰 + 계획서를 같이 근거**로 쓸 수 있고, 출처는 구분됩니다.
- 리뷰 → `(강대 컴퓨터프로그래밍 평점 5점 리뷰 참고)`
- 계획서 → `(가천대 가천인세미나 강의계획서 참고)`

> 🔍 **중간 발견 3**: `temp_pdf/` 의 81개 PDF가 **전부 같은 계획서 1종의 복사본**이었습니다.
> 파이프라인은 완성됐지만, 실효를 보려면 **서로 다른 실제 계획서**를 넣고 다시 돌리면 됩니다.

## 4. 전체 그림 (한눈에)

```
 [원본 데이터]                    [전처리]                  [인덱싱]              [서비스]

 6개 학교 CSV  ──build_dataset──▶ reviews_real.csv ─┐
                                                    ├─ingest─▶ 벡터DB(store/) ─▶ rag_vectordb.py
 temp_pdf PDF ──build_syllabi──▶ syllabi.csv ──────┘                              (질문→답변+출처)
```

핵심 파일 정리:

| 파일 | 역할 |
|------|------|
| `build_dataset.py` | 강의평 6개 CSV → 정제 통합 (995건) |
| `build_syllabi.py` | 강의계획서 PDF → 텍스트 청크 |
| `embeddings.py` | 문장 → 의미 벡터 (LSA / Gemini) |
| `vectorstore.py` | 벡터 저장 + 최근접 검색 (numpy) |
| `ingest.py` | 임베딩 → 벡터DB 적재 |
| `rag_vectordb.py` | RAG 본체 (검색→생성) ⭐ |
| `test_vectordb.py` | 자동 검증 (22개 통과) |

## 6. 한계 & 다음 단계

**지금의 한계**
- 임베딩이 오프라인 LSA(가벼운 대신 진짜 트랜스포머보단 덜 정밀) → Gemini 키로 업그레이드 가능

**다음 단계 후보**
1. `GEMINI_API_KEY` 연결 → 자연스러운 AI 문장 답변 + 진짜 의미 임베딩
2. **Streamlit 웹 UI** (채팅창 형태, 이미 설치돼 있음)
3. 강의계획서 청크에서 페이지 헤더 등 노이즈 정제 강화
4. 기획서 4.2 — 사용자가 강의평을 붙여넣으면 자동 정제하는 파서

## 7. 용어 미니 사전

| 용어 | 쉽게 |
|------|------|
| RAG | "오픈북 시험" 방식 AI — 자료 찾아서(검색) 답하기(생성) |
| 임베딩 | 문장을 의미 좌표(숫자)로 바꾸는 것 |
| 벡터DB | 그 좌표들을 저장하고 "가까운 것"을 빨리 찾는 창고 |
| TF-IDF | 글자/단어 빈도로 비슷함을 재는 고전 방식 (의미는 모름) |
| LSA | TF-IDF를 의미 축으로 압축한 가벼운 임베딩 |
| 환각(Hallucination) | AI가 모르는 걸 그럴듯하게 지어내는 현상 |
| 청킹(Chunking) | 긴 문서를 검색하기 좋게 조각내는 것 |
